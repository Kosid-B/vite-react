# Negotiation Leverage & Market Timing

**เป้าหมาย**: ก่อนเข้าโต๊ะเจรจาที่คุณดูเหมือน "ฝ่ายเสียเปรียบ" — สร้างอำนาจต่อรองขึ้นมาก่อน แล้วเลือกจังหวะเวลาให้เป็นตัวคูณมูลค่า เพื่อเปลี่ยนดีลที่เสียเปรียบให้เป็นธรรมหรือได้เปรียบ

> ที่มา: กรณีศึกษา Pixar × Steve Jobs (Toy Story, 1995) — สตูดิโอเกือบล้มละลายที่ติดสัญญา Disney เสียเปรียบ พลิกเกมด้วยการ IPO 1 สัปดาห์หลังหนังฉาย จนได้ทุน + อำนาจต่อรองไปเจรจาใหม่เป็น 50/50 และสุดท้าย Disney ซื้อ Pixar $7.4B

## A. หลักการหลัก (Core Principles)

- **BATNA — Best Alternative to a Negotiated Agreement**: อำนาจต่อรองไม่ได้มาจากคำพูดบนโต๊ะ แต่มาจาก "ทางเลือกที่ดีที่สุดถ้าดีลนี้ล่ม" ถ้าไม่มี → สร้างมันจากที่อื่น (เงินทุน ฐานลูกค้า ช่องทางรายได้สำรอง ผลงานที่พิสูจน์แล้ว) ก่อนกลับไปเจรจา
- **Leverage from Elsewhere**: เมื่อคู่เจรจาคุมทรัพยากรที่คุณต้องพึ่ง ให้หาแหล่งทรัพยากรทดแทนเพื่อ "ลดการพึ่งพา" — Jobs ใช้ตลาดหุ้น (IPO) แทนการพึ่งเงินทุนของ Disney
- **Market Timing**: จังหวะเวลาคือตัวคูณมูลค่า — เปิดตัว/ระดมทุน/ขึ้นราคา ให้ตรงกับช่วงที่คุณมี "หลักฐานความสำเร็จสด ๆ" อยู่ในมือ (Jobs จัด IPO 1 สัปดาห์หลังหนังฉายตอนกระแสพีค)
- **Crisis as Upgrade**: เมื่อโดน feedback แรง ๆ หรือของพัง อย่าตั้งรับ — ใช้มันรื้อให้ดีขึ้น เวอร์ชันหลังวิกฤตมักแข็งแรงกว่าเดิม (ทีมรื้อบท Woody หลัง Disney สั่งหยุดโปรเจกต์)
- **No Deal is Permanent**: สัญญาที่เสียเปรียบไม่ใช่จุดจบ — เก็บสะสม leverage แล้วกลับมาเจรจาใหม่เมื่อสมดุลอำนาจเปลี่ยน

## B. Workflow ก่อนเข้าโต๊ะเจรจา (Pre-Negotiation)

| ขั้นตอน | คำถามที่ต้องตอบ | ผลลัพธ์ |
|---|---|---|
| 1. Map Dependency | เราพึ่งอีกฝ่ายในเรื่องอะไร (ทุน/ช่องทาง/สิทธิ์)? | รายการจุดอ่อนที่ถูกจับเป็นตัวประกัน |
| 2. Build BATNA | ถ้าดีลล่ม ทางเลือกที่ดีที่สุดของเราคืออะไร? สร้างได้ที่ไหน? | แหล่ง leverage ทดแทน (ทุน/ลูกค้า/พันธมิตร) |
| 3. Find the Peak | ช่วงไหนที่เรามี "หลักฐานความสำเร็จ" ชัดที่สุด? | วันเวลาที่ควรกดปุ่ม (เปิดตัว/ระดมทุน/เจรจา) |
| 4. Reframe Crisis | จุดที่กำลังพัง จะใช้ยกคุณภาพได้อย่างไร? | แผนพลิกวิกฤตเป็นเวอร์ชันที่ดีขึ้น |
| 5. Renegotiate | เมื่อมี leverage แล้ว จะขอแก้เงื่อนไขข้อไหนก่อน? | เป้าหมายสัญญาใหม่ (เรียงตามความสำคัญ) |

## C. Checklist ก่อนกดปุ่ม

- [ ] เรามี BATNA จริงหรือแค่หวังว่าอีกฝ่ายจะใจดี?
- [ ] จังหวะเวลานี้เราถือหลักฐานความสำเร็จอะไรในมือบ้าง (ยอดขาย/ผู้ใช้/ดีลที่ปิดแล้ว)?
- [ ] ถ้าเดินออกจากโต๊ะตอนนี้ เราอยู่รอดกี่เดือน? (runway)
- [ ] เงื่อนไข 3 ข้อที่เสียเปรียบที่สุดในสัญญาปัจจุบันคืออะไร?
- [ ] มี "ทางหนี" ให้อีกฝ่ายรักษาหน้าไหม (win-win exit)?

## D. สคริปต์เปิดเจรจา (ตัวอย่าง)

> "เราเห็นคุณค่าของความร่วมมือนี้มาก และอยากเดินต่อระยะยาว — ช่วงที่ผ่านมา [หลักฐานความสำเร็จ: ยอด/ผู้ใช้/ดีล] ทำให้บริบทเปลี่ยนไปจากตอนเซ็นสัญญาเดิม ผมจึงอยากทบทวนเงื่อนไข [ข้อที่ต้องการแก้] ให้สะท้อนสถานการณ์ปัจจุบัน ถ้าตกลงกันได้ เราพร้อมขยายความร่วมมือใน [สิ่งที่อีกฝ่ายได้เพิ่ม]"

## E. Decision Gate

- **อย่าเจรจาตอนอ่อนแอที่สุด**: ถ้ายังไม่มี BATNA และไม่มีหลักฐานความสำเร็จ → เลื่อนการเจรจา ไปสร้าง leverage ก่อน (อย่ารีบเซ็น)
- **Ethical guardrail**: สร้างอำนาจต่อรองจาก "คุณค่าจริง" (ทุน ผลงาน ทางเลือก) ไม่ใช่การหลอก/ข่มขู่/ปั่นข้อมูลเท็จ — win-win ที่ยั่งยืนชนะ win-lose ที่ครั้งเดียว

## การเชื่อมกับเครื่องมือในระบบ CEO AI Thailand

- **หา leverage จากรายได้**: ใช้หน้า "เมืองบริษัท → คลังเมือง" (finance) ดู runway จริงก่อนกำหนดจุดยืนเจรจา
- **Market Timing**: ใช้คู่กับ Skill "Market Insight & Strategy (Thailand)" เลือกจังหวะ/ฤดูกาลที่กำลังซื้อพีค
- **หลักฐานความสำเร็จ**: ดึงจาก SaaS Analytics / รายงานผู้บริหาร (PDF) ไปวางบนโต๊ะเจรจา
- **B2B (RFQ)**: ใช้กรอบนี้ตั้งราคา/ต่อรองในดีล RFQ ระหว่างคู่ค้าในตลาด
