# Customer-Centric Disruption & Category Design

**เป้าหมาย**: หา "ช่องตีตลาด" ที่เจ้าตลาดมองข้าม โดยจับปัญหาที่ลูกค้าถูกสั่งให้ทน (แต่ถูกเรียกว่า "คาแรคเตอร์") มาสร้างเป็นหมวดหมู่สินค้าใหม่ที่ทั้งแก้ปัญหาและเก่งกว่า จนแย่งมงกุฎจากเจ้าตลาดได้

> ที่มา: กรณีศึกษา Honda NSX (1990) — ยุค 80 Super Car ถูกผูกขาดโดย Ferrari/Lamborghini ที่ขับยาก แอร์ไม่เย็น โอเวอร์ฮีตง่าย โดยค่ายยุโรปบอกว่า "นี่คือคาแรคเตอร์ ถ้าขับไม่ไหวคุณไม่คู่ควร" Honda มองเป็น "ปัญหาลูกค้าที่ยังไม่มีใครแก้" จึงสร้าง NSX (แชสซีอะลูมิเนียมทั้งคันคันแรกของโลก + VTEC ลาก 8,000 รอบ + Ayrton Senna จูนช่วงล่าง) ที่เร็วกว่า เบากว่า เข้าโค้งคมกว่า Ferrari 348 แต่ขับง่ายเหมือน Civic = สร้างหมวดหมู่ "Everyday Super Car" จน Ferrari ต้องกลับไปรื้อรถทั้งค่าย

## A. หลักการหลัก (Core Principles)

- **"คาแรคเตอร์" = Pain Point ที่ถูก reframe**: เมื่อเจ้าตลาดบอกว่า "มันก็เป็นแบบนี้แหละ ต้องทน" นั่นคือสัญญาณว่ามีปัญหาที่ยังไม่มีใครกล้าแก้ — เพราะแก้แล้วต้องลงทุนหนัก เจ้าตลาดเลยเลือก "ตีกรอบให้มันเป็นเรื่องปกติ" แทน
- **Category Design (Everyday X)**: อย่าแข่งในหมวดเดิมด้วยกติกาเดิม — สร้างหมวดใหม่ที่คุณนิยามเอง เช่น "Everyday Super Car" = เอาสมรรถนะระดับพรีเมียมมาทำให้ใช้ได้จริงทุกวัน คนจะเทียบคุณกับ "ความคาดหวังใหม่" ไม่ใช่กับเจ้าตลาด
- **"ได้ทั้งคู่" คือกำแพงกันลอก (Both/And, not Either/Or)**: จุดที่ยากคือ NSX ไม่ยอมแลกสมรรถนะเพื่อความใช้ง่าย แต่ทำทั้งสองอย่างพร้อมกัน — สิ่งที่คู่แข่งลอกยากคือ "การทำสองสิ่งที่ดูขัดกันให้อยู่ในของชิ้นเดียว"
- **Proof by Insider (ดึงตัวจริงมายกระดับ)**: Senna จูนช่วงล่าง = ความน่าเชื่อถือที่เงินโฆษณาซื้อไม่ได้ ผู้เชี่ยวชาญ/ผู้ใช้ตัวจริงเปลี่ยนสินค้าที่ดีให้กลายเป็นตำนานที่คนเล่าต่อ
- **ความหยิ่งของเจ้าตลาดคือโอกาสของคุณ**: ยิ่งเจ้าตลาดภูมิใจใน "คาแรคเตอร์" มากเท่าไร ยิ่งช้าในการยอมรับว่ามันคือปัญหา = ช่องให้คุณเข้าไปก่อน

## B. Workflow ออกแบบสินค้าท้าเจ้าตลาด (Pre-Launch)

| ขั้นตอน | คำถามที่ต้องตอบ | ผลลัพธ์ |
|---|---|---|
| 1. Pain-Point Audit | ลูกค้าถูกสั่งให้ "ทน" อะไรบ้าง ที่วงการเรียกว่าคาแรคเตอร์/ธรรมเนียม? | รายการ pain point ที่ถูก normalize |
| 2. Pick the "Un-fixable" | อันไหนที่เจ้าตลาด "แก้ได้แต่ไม่ยอมแก้" (เพราะแพง/ขัดอีโก้)? | 1–2 pain point ที่เป็นช่องตีจริง |
| 3. Both/And Test | จะแก้ปัญหานี้ **โดยไม่ลด** จุดแข็งหลักของหมวดได้อย่างไร? | สเปกสินค้าที่ "ได้ทั้งคู่" |
| 4. Name the Category | ถ้าตั้งชื่อหมวดใหม่ให้สิ่งนี้ จะชื่ออะไร (Everyday X / X for humans)? | ชื่อหมวด + นิยาม + ความคาดหวังใหม่ |
| 5. Recruit an Insider | ใครคือ "Senna" ของวงการเราที่มารับรอง/ร่วมออกแบบได้? | ผู้เชี่ยวชาญ/ผู้ใช้ตัวจริงที่ยกระดับความน่าเชื่อถือ |
| 6. Aim the Message | จะสื่อสารว่า "ของเดิมทำให้คุณลำบากโดยไม่จำเป็น" อย่างไรให้ไม่ดูถูกลูกค้า? | ข้อความเปิดตัวที่ปลดล็อกความอึดอัดเดิม |

## C. Category Design Canvas

```
หมวดเดิม (เจ้าตลาดครอง):  ____________________
กติกาที่เจ้าตลาดตั้ง:       ____________________
Pain point ที่ถูก normalize: ____________________
หมวดใหม่ที่เรานิยาม:        Everyday ______________
สิ่งที่เรา "ได้ทั้งคู่":      ______ + ______ (ที่ปกติต้องเลือกอย่างเดียว)
Insider ที่มารับรอง:        ____________________
ความคาดหวังใหม่ของลูกค้า:   ____________________
```

## D. Checklist ก่อนออกสินค้า

- [ ] pain point ที่เราแก้ เป็นเรื่องที่ลูกค้า "ทนมานาน" จริงไหม (ไม่ใช่เราคิดไปเอง)?
- [ ] เราแก้มันโดย **ไม่ลด** จุดแข็งหลักของหมวดใช่ไหม (Both/And ไม่ใช่ trade-off)?
- [ ] "การทำได้ทั้งคู่" ของเรา คู่แข่งลอกยากแค่ไหน (กำแพงคืออะไร: เทคโนโลยี/ต้นทุน/ความสัมพันธ์)?
- [ ] เรามีชื่อหมวดใหม่ + นิยามที่ลูกค้าเข้าใจใน 1 ประโยคไหม?
- [ ] มี insider/ผู้ใช้ตัวจริงมารับรองหรือร่วมออกแบบหรือยัง?
- [ ] ข้อความเปิดตัว "ปลดความอึดอัดเดิม" โดยไม่ทำให้ลูกค้าเจ้าตลาดรู้สึกโง่ใช่ไหม?

## E. กับดักที่พบบ่อย (Anti-patterns)

- **ลดราคาแทนที่จะแก้ปัญหา** — การเป็น "ของถูกกว่า" ไม่ใช่ category design มันคือการยอมแพ้ในหมวดเดิม NSX ไม่ได้ถูกกว่า Ferrari มันแก้ปัญหาที่ Ferrari ไม่ยอมแก้
- **แก้ pain point แล้วเสียจุดแข็งหลัก** — ถ้าทำให้ใช้ง่ายแต่ช้าลง/ห่วยลง = ได้แค่ของกลาง ๆ ที่ไม่มีใครตื่นเต้น
- **ประกาศหมวดใหม่แต่ของไม่ถึง** — category design ต้องมี "ของจริง" หนุน ไม่ใช่แค่การตลาด NSX มีอะลูมิเนียม + VTEC + Senna จริง
- **ดูถูกลูกค้าเจ้าตลาด** — สื่อสารว่า "คนที่ใช้ของเดิมโง่" = สร้างศัตรู ควรสื่อว่า "คุณไม่จำเป็นต้องทนอีกต่อไป"

## F. นำไปใช้กับ CEO AI Thailand / SME ไทย

- **SaaS/บริการที่ลูกค้า "ถูกสั่งให้ทน"**: ระบบ ERP/บัญชีที่ "มันก็ยากแบบนี้แหละ ต้องอบรม 3 วัน" → โอกาสทำ "Everyday ERP" ที่ใช้ได้ใน 10 นาที
- **ตลาด/marketplace**: ถ้าเจ้าตลาดบอกว่า "ค่าธรรมเนียมแพงเพราะเราพรีเมียม" → ช่องทำแพลตฟอร์มที่ทั้งถูกกว่า **และ** บริการดีกว่า (Both/And)
- **Insider ไทย**: ดึงผู้เชี่ยวชาญ/อินฟลูฯ สายวิชาชีพตัวจริง (ไม่ใช่ดารา) มาร่วมออกแบบ/รับรอง = ความน่าเชื่อถือแบบ Senna
- **ผูกกับ Pulse & A/B**: ทดสอบข้อความ "ปลดความอึดอัดเดิม" กับกลุ่มเป้าหมายก่อน scale
