## หลักการวัดและทบทวน (6.2 + เชื่อม 9.1)
- **วิธีวัด:** ใช้หลักฐานเชิงระบบเป็นหลัก (query สิทธิ์จริง, CI log, advisor, backup log) ไม่ใช่ความรู้สึก — ตามบทเรียน "verify ไม่ใช่เดา" ([[R19]])
- **บันทึกผล:** เก็บผลการวัดแต่ละไตรมาสเป็น documented information (input ของ Management Review 9.3)
- **เกณฑ์ทบทวน:** ถ้าตัวชี้วัดไม่บรรลุ 2 รอบติด → เปิดเป็น nonconformity + corrective action (10.2)

## การสื่อสาร (6.2 requirement)
- วัตถุประสงค์นี้สื่อสารต่อผู้เกี่ยวข้องผ่านเอกสาร ISMS + ทบทวนในการประชุมทบทวนฝ่ายบริหาร
- ปัจจุบันทีมเล็ก (ผู้ปฏิบัติงานหลักคนเดียว) — เมื่อขยายทีมต้องสื่อสาร+อบรมตาม O9
