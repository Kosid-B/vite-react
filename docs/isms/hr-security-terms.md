# ข้อกำหนดความมั่นคงในการจ้างงานและข้อตกลงรักษาความลับ (NDA)
**CEO AI Thailand — ISO/IEC 27001:2022 controls 6.2, 6.6**

| | |
|---|---|
| เวอร์ชัน | 1.0 |
| วันที่ | 2026-07-06 |
| ใช้กับ | พนักงาน ผู้รับเหมา ฟรีแลนซ์ และผู้ร่วมงานทุกคนที่เข้าถึงระบบ/ข้อมูล |

> **บริบท:** ปัจจุบันธุรกิจขนาดเล็ก (ผู้ปฏิบัติงานหลักคนเดียว) แต่เอกสารนี้เตรียมไว้ล่วงหน้าสำหรับ
> เมื่อมีการจ้างทีม/ผู้รับเหมา — เป็นข้อกำหนดที่ certification body คาดหวังให้มีพร้อมก่อนขยายทีม

## 1. ข้อกำหนดความมั่นคงในเงื่อนไขการจ้าง (control 6.2)
สัญญาจ้าง/สัญญาบริการทุกฉบับต้องระบุอย่างน้อย:
1. หน้าที่รักษาความลับของข้อมูล (ทั้งระหว่างและหลังสิ้นสุดการจ้าง)
2. ความรับผิดชอบในการปฏิบัติตามนโยบายความมั่นคงสารสนเทศของธุรกิจ
3. การใช้สินทรัพย์/บัญชี/สิทธิ์เข้าถึงเฉพาะเท่าที่จำเป็นต่องาน
4. หน้าที่รายงานเหตุการณ์ความมั่นคง (control 6.8) และผลของการฝ่าฝืน (กระบวนการทางวินัย 6.4)
5. การคืนสินทรัพย์และเพิกถอนสิทธิ์เมื่อสิ้นสุดการจ้าง (control 5.11, 6.5)

## 2. ข้อตกลงรักษาความลับ (NDA) — โครงร่างมาตรฐาน (control 6.6)

**คู่สัญญา:** CEO AI Thailand (ผู้เปิดเผย) และ ผู้รับข้อมูล (พนักงาน/ผู้รับเหมา)

**1. นิยาม "ข้อมูลลับ"** — รวมถึงแต่ไม่จำกัดเพียง: source code, คีย์ API/credential, ข้อมูลลูกค้าและผู้ใช้,
ข้อมูลธุรกิจ/แผนกลยุทธ์, schema ฐานข้อมูล, ข้อมูลการเงิน และข้อมูลใด ๆ ที่ระบุว่าเป็นความลับหรือควรเข้าใจได้ว่าเป็นความลับ

**2. หน้าที่** — ผู้รับข้อมูลต้อง: (ก) เก็บรักษาเป็นความลับ (ข) ใช้เฉพาะเพื่อปฏิบัติงานที่ได้รับมอบหมาย
(ค) ไม่เปิดเผยต่อบุคคลที่สาม (ง) ปกป้องด้วยมาตรการที่สมเหตุสมผลไม่ต่ำกว่าที่ใช้กับข้อมูลลับของตนเอง

**3. ข้อยกเว้น** — ข้อมูลที่เป็นสาธารณะอยู่แล้วโดยไม่ใช่ความผิดของผู้รับ, ที่ทราบอยู่ก่อนโดยชอบ,
หรือที่ต้องเปิดเผยตามกฎหมาย (โดยแจ้งผู้เปิดเผยล่วงหน้าเท่าที่ทำได้)

**4. ระยะเวลา** — หน้าที่รักษาความลับมีผลต่อเนื่อง **หลังสิ้นสุดความสัมพันธ์อย่างน้อย [ระบุ เช่น 3 ปี]**
(สำหรับความลับทางการค้า — ตลอดไปตราบเท่าที่ยังเป็นความลับ)

**5. การคืน/ทำลายข้อมูล** — เมื่อสิ้นสุดการจ้าง ผู้รับต้องคืนหรือทำลายข้อมูลลับทั้งหมด (สอดคล้อง 5.11, 7.14)

**6. ผลของการฝ่าฝืน** — [ระบุ] และธุรกิจมีสิทธิดำเนินการตามกฎหมาย

_ลงชื่อ ผู้รับข้อมูล ______________ วันที่ ______  /  ผู้แทนธุรกิจ ______________ วันที่ _______

> ⚠️ **หมายเหตุ:** นี่คือ**โครงร่าง**เพื่อความครบถ้วนของ control ไม่ใช่คำแนะนำทางกฎหมาย
> ควรให้ทนายตรวจก่อนใช้จริง เพื่อให้สอดคล้องกฎหมายไทย (PDPA, กฎหมายแรงงาน)

## ความเชื่อมโยง
- **6.5** (ความรับผิดชอบหลังพ้นสภาพ), **5.11** (คืนสินทรัพย์), **7.14** (ทำลายข้อมูลในอุปกรณ์คืน)
- **6.4** (กระบวนการทางวินัย) — ผลของการฝ่าฝืน NDA
- **6.1** (screening) — ยังเป็นช่องว่างแยก ควรมีกระบวนการตรวจสอบก่อนจ้างเมื่อรับทีม
