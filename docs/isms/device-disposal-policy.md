## บริบท

ถ้ากำจัด/ขาย/คืนเครื่องโดยไม่ล้างข้อมูลก่อน มีความเสี่ยงที่คีย์เก่าจะกู้คืนได้แม้จะลบไฟล์ไปแล้ว (การลบไฟล์ปกติไม่ได้ล้างข้อมูลจริงจากดิสก์)

## ระดับการล้างข้อมูล (ตาม ISO/IEC 27040 / สอดคล้อง NIST SP 800-88)

| ระดับ | วิธีการ | ใช้เมื่อ |
|---|---|---|
| **Clear** | Overwrite ข้อมูลด้วยวิธีทั่วไป (เช่น full-disk format ที่เขียนทับ, `cipher /w` บน Windows) | ก่อน**ใช้ต่อภายในองค์กร** (มอบเครื่องให้พนักงานคนอื่น) |
| **Purge** | ล้างเชิงลึก — cryptographic erase (SSD ที่เข้ารหัส disk เต็มไดรฟ์), หรือ secure erase ผ่านเครื่องมือเฉพาะ | ก่อน**ขาย/คืนเครื่องเช่า/บริจาค** ให้บุคคลภายนอก |
| **Destruct** | ทำลายสื่อทางกายภาพ (เจาะ/บด/เผาแผ่นดิสก์) | อุปกรณ์เสีย/หมดอายุ ที่ไม่สามารถ purge ได้ หรือเก็บข้อมูลอ่อนไหวสูงสุด |

## ขั้นตอนก่อนกำจัด/ขาย/คืนอุปกรณ์ (บังคับ)

1. **ตรวจสอบก่อนว่าอุปกรณ์นี้เคย/กำลังเก็บความลับอะไรบ้าง**
   - ไฟล์คีย์/credential (เช่นโฟลเดอร์ `CEO-Ai-Thailand-secrets`)
   - `.env` ที่เคยมี Supabase/Cloudflare/Anthropic key
   - Git credential / SSH key ที่ผูกกับ repo
   - Browser saved password/session
2. **หมุนคีย์ทุกตัวที่เครื่องนี้เคยเก็บไว้ก่อน** (ดู R1 — ไม่พึ่งแค่การล้างเครื่องอย่างเดียว เพราะไม่รู้ว่าเคยรั่วไปที่อื่นหรือไม่)
3. **เลือกระดับการล้างข้อมูลให้ตรงกับปลายทาง** (ดูตารางด้านบน)
4. **บันทึก sanitization record** — วันที่, อุปกรณ์ (serial/asset tag), วิธีที่ใช้, ผู้ดำเนินการ, ปลายทาง (ใช้ต่อ/ขาย/ทำลาย)
5. เก็บบันทึกนี้ไว้เป็นหลักฐานตรวจประเมิน (audit evidence) อย่างน้อย 1 ปี
