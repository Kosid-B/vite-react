# Concierge Seed Checklist — จับคู่ดีลแรกด้วยมือ (คู่ขนานกับระบบ)

> จาก action plan `B2B-SELLER-DEEPDIVE-ACTIONPLAN.md` ข้อ 6: marketplace ใหม่ตายเพราะ cold-start.
> ระบบทำ single-player value + seed listing แล้ว (โค้ด) — แต่ **"ดีลปิดใบแรก + รีวิวจริง" ต้องเกิดเร็ว**
> เพื่อให้ล้อ รีวิว→SEO→organic หมุน. งานนี้ทำ **ด้วยมือ** โดยทีม B. Training (ใช้เครือข่ายที่ปรึกษา 20 ปี).
> เป้าหมาย: **ดีลปิดสำเร็จ 5–10 ดีลแรก + รีวิวจริง ≥5** ภายใน 30–45 วัน แล้วปล่อยให้ organic รับช่วง.

## หลักการ (ทำไมทำมือช่วงแรกคุ้ม)
- Marketplace ต้องข้าม "ดีลแรก" ที่พิสูจน์ว่า **ระบบใช้ปิดงานได้จริง** ก่อน organic จะเชื่อ
- ทำมือ = เรียนรู้ friction จริง (ผู้ขาย/ผู้ซื้อสะดุดตรงไหน) → ป้อนกลับให้ทีมพัฒนา
- ทุกดีลที่ปิด = 1 รีวิวจริง → `AggregateRating` ขึ้น Google → ดึง organic รอบใหม่ (ไม่ใช่รีวิวปลอม)

## Checklist — 3 เฟส

### เฟส 1 · Seed Supply (สัปดาห์ 1–2) — ให้ /b มีร้านจริง
- [ ] คัด **10–15 ที่ปรึกษา/ผู้ให้บริการ** จากเครือข่าย (ISO, อบรม, การตลาด, บัญชี, กฎหมาย)
- [ ] ช่วยแต่ละรายเปิดหน้าร้านผ่าน onboarding <5 นาที (`หน้าร้านของฉัน` → auto-draft → AI เขียน VP → เผยแพร่)
- [ ] ตรวจว่าร้านขึ้นในสารบัญ `/b` ถูกหมวด DBD + มี VP/บริการครบ
- [ ] เป้า: **≥10 ร้าน published** ที่เป็นธุรกิจจริง (ไม่ใช่ร้านทดสอบ)

### เฟส 2 · จุดชนวน Demand + จับคู่ (สัปดาห์ 2–4) — ให้เกิด RFQ จริง
- [ ] หาผู้ซื้อจริง 5–10 ราย (SME ในเครือข่ายที่มีงานต้องจ้าง เช่น ทำ ISO, อบรม, การตลาด)
- [ ] ช่วยผู้ซื้อโพสต์ **ประกาศงานกลาง (Open RFQ)** — ใช้ปุ่ม "✨ AI ช่วยร่าง" ให้ร่างเร็ว
- [ ] จับคู่ด้วยมือ: แจ้งผู้ขายที่ตรงหมวดให้เข้ามา claim + เสนอราคา (ใช้ "✨ AI ร่างใบเสนอราคา")
- [ ] ประกบจนผู้ซื้อ **กดรับใบเสนอราคา → เกิดออเดอร์** (สถานะ accepted)
- [ ] เป้า: **≥8 ออเดอร์** ถูกสร้าง

### เฟส 3 · ปิดดีล + เก็บรีวิว (สัปดาห์ 3–6) — หมุนล้อ รีวิว→SEO
- [ ] ติดตามให้ผู้ขายส่งมอบงาน → อัปเดตสถานะออเดอร์เป็น `delivered` → `completed`
- [ ] เมื่อ completed: เชิญผู้ซื้อรีวิว (ระบบเปิดฟอร์มรีวิวเฉพาะออเดอร์ที่ปิดจริงเท่านั้น — RLS 0032)
- [ ] เป้า: **ดีลปิดสำเร็จ ≥5 + รีวิวจริง ≥5** → `AggregateRating` เริ่มโชว์บน `/b` และ Google
- [ ] ส่ง `sitemap.xml` เข้า Google Search Console + Rich Results Test 1 หน้าร้านที่มีรีวิว

## วัดผล (ดูที่ไหน)
- **หน้า ซื้อขาย B2B (`trade`)** → พาเนล "📊 สุขภาพดีลของฉัน" (RFQ→ใบเสนอราคา→รับ→ปิดดีล)
- **GA4** events: `open_rfq_posted`, `open_rfq_shared`, `rfq_quote_sent`, `rfq_from_outsource`,
  `seller_onboard_step`, `storefront_published`
- เมื่อ **ดีลปิด/เดือน สม่ำเสมอ** (เช่น ≥10/เดือน 2 เดือนติด) → พิจารณาเปิด take-rate 3% (Xendit) และลดการจับคู่ด้วยมือ

## กติกาสำคัญ (ความซื่อสัตย์)
- ❌ **ห้ามสร้างรีวิวปลอม/ร้านปลอม** — รีวิวต้องมาจากออเดอร์ที่ปิดจริง (บังคับด้วย RLS ที่ DB)
- ❌ **ยังไม่เปิดค่าฟี 3%** จนกว่าดีลปิดจะสม่ำเสมอ — ช่วง seed ฟรีเพื่อสร้างพฤติกรรม
- ✅ ร้าน seed ของ B. Training (`STARTER_LISTINGS`) ติดป้ายชัดว่าเป็นบริการของผู้พัฒนา (โปร่งใส)
