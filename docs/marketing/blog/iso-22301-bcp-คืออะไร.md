# ISO 22301 & BCP คืออะไร? แผนความต่อเนื่องธุรกิจที่ SME/โรงงานต้องมี

> **สรุปสั้น:** ISO 22301 คือมาตรฐานระบบบริหารความต่อเนื่องทางธุรกิจ (BCMS) — ช่วยให้ธุรกิจ "ไม่หยุดชะงัก"
> เมื่อเจอวิกฤต (ไฟไหม้ น้ำท่วม ระบบล่ม ซัพพลายขาด) หัวใจคือ **BIA** (วิเคราะห์ผลกระทบ) + **BCP** (แผนกู้คืน)

น้ำท่วมนิคม ไฟไหม้โรงงาน ระบบ IT ล่ม ซัพพลายเออร์เจ๊ากะทันหัน — เหตุการณ์พวกนี้ทำให้ธุรกิจที่ "กำไรดี"
ต้องหยุดยาวจนเสียลูกค้าถาวร **ISO 22301** คือระบบที่เตรียมให้ธุรกิจ **ล้มแล้วลุกเร็ว** โดยเฉพาะโรงงาน/SME
ในเขตอุตสาหกรรมที่ลูกค้า (โดยเฉพาะลูกค้าต่างชาติ/ยานยนต์) มักขอให้คู่ค้ามี BCP

## ISO 22301 vs BCP ต่างกันยังไง

- **BCP (Business Continuity Plan)** = แผนกู้คืนธุรกิจ (เอกสาร/ขั้นตอนว่าเมื่อเกิดเหตุจะทำอะไร ใครรับผิดชอบ)
- **ISO 22301** = มาตรฐานระบบทั้งหมด (BCMS) ที่ครอบ BCP อีกที — มีการวิเคราะห์ วางแผน ซ้อม ทบทวน ต่อเนื่อง

พูดง่าย ๆ: BCP คือ "แผน" · ISO 22301 คือ "ระบบที่ทำให้แผนนั้นใช้ได้จริงและอัปเดตเสมอ"

## หัวใจของ ISO 22301: BIA + กลยุทธ์ + แผน

| ขั้น | ทำอะไร | ผลลัพธ์ |
|---|---|---|
| **1. BIA** (Business Impact Analysis) | หากระบวนการสำคัญ + ผลกระทบถ้าหยุด | รู้ว่าอะไร "ห้ามหยุด" และหยุดได้นานสุดกี่ชั่วโมง (RTO) |
| **2. ประเมินความเสี่ยง** | ภัยอะไรจะทำให้หยุด (ไฟ/น้ำ/ไซเบอร์/ซัพพลาย) | จัดลำดับความเสี่ยงที่ต้องเตรียม |
| **3. กลยุทธ์ความต่อเนื่อง** | จะกู้คืนยังไง (สำรองไฟ/สถานที่/ซัพพลายเออร์สำรอง) | ทางเลือกเมื่อเกิดเหตุ |
| **4. เขียน BCP** | ขั้นตอน + ทีม + ช่องทางสื่อสาร | เอกสารพร้อมใช้ |
| **5. ซ้อม + ทบทวน** | ซ้อมแผน (drill) อย่างน้อยปีละครั้ง | แผนที่ใช้ได้จริง ไม่ใช่แค่กระดาษ |

> **RTO (Recovery Time Objective)** = เวลาที่ยอมให้กระบวนการหยุดได้สูงสุดก่อนกระทบธุรกิจหนัก — ตัวเลขนี้คือหัวใจ
> ของ BIA ที่ทำให้ตัดสินใจได้ว่าต้องลงทุนสำรองอะไรบ้าง

## ทำไมโรงงาน/SME เขต EEC ควรมี ISO 22301

- **ลูกค้าองค์กร/ต่างชาติมักบังคับ** ให้ซัพพลายเออร์มี BCP (โดยเฉพาะยานยนต์/อิเล็กทรอนิกส์/อาหารส่งออก)
- **ประกันภัยธุรกิจ** พิจารณาเบี้ยดีขึ้นเมื่อมีระบบความต่อเนื่อง
- **เขต EEC เสี่ยงภัยเฉพาะ** (น้ำท่วม ไฟฟ้า ซัพพลายเชน) — เตรียมไว้ = ได้เปรียบคู่แข่งเมื่อเกิดเหตุจริง

## ✅ เช็กลิสต์เริ่มต้น ISO 22301

- [ ] ระบุกระบวนการสำคัญ (critical) ที่ห้ามหยุด
- [ ] ทำ BIA — กำหนด RTO ของแต่ละกระบวนการ
- [ ] ประเมินความเสี่ยงหลัก (ไฟ/น้ำ/ไซเบอร์/ซัพพลาย/คน)
- [ ] กำหนดกลยุทธ์กู้คืน + ทรัพยากรสำรอง
- [ ] เขียน BCP + รายชื่อทีม + ช่องทางสื่อสารฉุกเฉิน
- [ ] ซ้อมแผน (tabletop/drill) อย่างน้อยปีละ 1 ครั้ง + บันทึกผล
- [ ] ทบทวนโดยฝ่ายบริหาร + ปรับปรุง

## FAQ

**Q: SME เล็กจำเป็นต้องมี ISO 22301 ไหม?**
A: ถ้าลูกค้าไม่ได้บังคับ อาจเริ่มจาก BCP อย่างง่ายก่อนก็ได้ แต่ถ้าเป็นซัพพลายเออร์ในห่วงโซ่ส่งออก/ยานยนต์ แนะนำให้มี

**Q: ISO 22301 กับ ISO 9001 ทำพร้อมกันได้ไหม?**
A: ได้ และแนะนำ เพราะใช้โครงสร้าง Annex SL ร่วมกัน (ข้อ 4–10) ลดงานซ้ำซ้อน

**Q: ต้องซ้อมแผนบ่อยแค่ไหน?**
A: อย่างน้อยปีละ 1 ครั้ง และทุกครั้งที่มีการเปลี่ยนแปลงสำคัญ (ย้ายโรงงาน/เพิ่มสายการผลิต)
