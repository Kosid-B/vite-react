# ISO Readiness Pilot — ข้อเสนอเสียเงินทดลอง (Paid Pilot) สำหรับ beachhead ISO/มอก.

> **โมเดล:** Hybrid A+C (ยืนยันโดย Board) — เก็บ free trial (A) ไว้เป็นปากทางฟันเนล
> + เพิ่ม **Paid Pilot (C)** เป็น "ข้อเสนอปิดการขาย" สำหรับ outreach กลุ่ม ISO/มอก. ที่อุ่นและเจ็บเรื่อง audit
> **เป้าหมาย:** ได้ลูกค้า **จ่ายเงินรายแรก** เร็วสุด โดยไม่ทำลาย funnel ฟรีของคนเย็น

---

## 1) ทำไมต้องมี Pilot (ไม่ใช่แค่ free trial)
| | Free trial (A) | ISO Pilot (C) |
|---|---|---|
| กลุ่ม | คนเย็น (organic/Direct) | คนอุ่น (outreach ISO) |
| friction | ต่ำ (ลองเอง) | มี (จ่ายเงิน) = **qualify คนจริงจัง** |
| ได้เงิน | ไม่ได้ | **ทันที** |
| คุณค่าที่ให้ | ลองระบบเอง | **ที่ปรึกษาช่วยลงมือ + ประเมินจริง** (done-with-you) |

คน SME/โรงงานที่ audit ใกล้เข้ามา **ไม่อยากลองเอง** — เขาอยากให้คน 20 ปีช่วยดูให้ = ยอมจ่ายเพื่อความมั่นใจ

---

## 2) รายละเอียดข้อเสนอ (offer)
**ชื่อ:** ISO Readiness Pilot
**ราคา:** **฿1,990 จ่ายครั้งเดียว** (ปรับได้ใน `config.ts` → `PAYMENT.pilotPrice`)
**สิ่งที่ลูกค้าได้:**
1. **Concierge ประเมินความพร้อม ISO/มอก.** โดยตรง (โทร/วิดีโอ ~30–45 นาที) — บอกพร้อมกี่ % + ขาดเอกสารอะไรก่อน audit
2. **ใช้ระบบ CEO AI Thailand เต็ม (Scale) 1 เดือน** — ทำเอกสาร/เช็กลิสต์/ติดตามผลต่อเอง
3. **รายงานสรุปช่องโหว่ + แผนปิด NC** ส่งเป็นไฟล์

**ทำไมราคานี้:** ต่ำกว่าค่าที่ปรึกษาเต็ม (หลักหมื่น–แสน) มาก แต่สูงพอให้ qualify คนจริงจัง
และ **anchor ให้ Growth ฿1,490/เดือน ดูคุ้ม** (Pilot จบแล้วชวนต่อ Growth รายเดือน)

**เส้นทางหลัง Pilot:** จบ 1 เดือน → เสนอต่อ **Growth ฿1,490/เดือน** (ทำ ISO ต่อเนื่อง) = แปลง one-time → recurring

---

## 3) การตั้งค่าเทคนิค (สิ่งที่ต้องทำ)
โค้ดพร้อมแล้ว — เหลือแค่สร้าง Stripe link + วางใน config:
1. Stripe Dashboard → **Payment Links → สร้างลิงก์ใหม่ (one-time ฿1,990)** เปิดบัตร + PromptPay
2. คัดลอก URL → วางใน `src/config.ts` → `PAYMENT.stripePaymentLinkPilot = 'https://buy.stripe.com/...'`
3. เมื่อวางแล้ว → **การ์ด Pilot จะโผล่อัตโนมัติ**ในหน้า `/pricing` (ตอนนี้ว่าง = ซ่อน ไม่มีปุ่มพัง)
4. webhook เดิม (`stripe-webhook`) รับ `checkout.session.completed` ของ Pilot ได้อยู่แล้ว (verify ผ่าน 200)

> ⚠️ Pilot เป็น one-time ไม่ใช่ subscription → ไม่ auto-upgrade plan ในระบบ
> ให้ทีมตั้ง Scale ให้ลูกค้า Pilot ด้วยมือ (หรือใช้ admin) 1 เดือน — เป็นงาน concierge อยู่แล้ว

---

## 4) ใช้ Pilot ใน outreach ยังไง (ผูกกับ Promotion Engine)
- **อย่าเปิดด้วยการขาย Pilot** — เปิดด้วย "ลองฟรี/ให้คุณค่าก่อน" (free) ตาม TRACK A
- คนที่ **ตอบรับ + เจ็บเรื่อง audit ชัด** → เสนอ Pilot: *"ถ้าอยากให้ผมช่วยดูระบบให้ตรงเลย มีแพ็ก Pilot ฿1,990 ประเมินให้ + ใช้ระบบ 1 เดือน"*
- ใส่ผลใน `outreach-tracker.csv` คอลัมน์ "แพ็กเกจที่เสนอ" = Pilot

**สคริปต์เสนอ Pilot (LINE/โทร):**
```
จากที่คุยกัน ผมว่าเคสของพี่เตรียม audit ได้เลยครับ
เรามีแพ็ก ISO Readiness Pilot — ผมช่วยประเมินให้ว่าโรงงานพร้อมกี่ % ขาดเอกสารอะไร
+ ให้ใช้ระบบเต็ม 1 เดือน ราคา 1,990 จ่ายครั้งเดียว
ถ้าพร้อมผมส่งลิงก์จ่ายให้เลยครับ เริ่มได้ทันที 🙏
```

## 5) วัดผล
- **North Star:** จำนวน Pilot ที่ขายได้/เดือน · เป้าเดือนแรก = **1–2**
- แปลง Pilot → Growth รายเดือน (retention) = ตัวชี้วัดว่าคุณค่าจริง
- ทุกลิงก์ Pilot ติด `utm_campaign=iso_beachhead` → เทียบใน GA4

> Pilot ไม่ได้แทน free trial — มันคือ "ประตูจ่ายเงิน" สำหรับคนที่พร้อมจ่ายอยู่แล้ว
> ปล่อยคนเย็นไหลผ่าน free trial ต่อไป · เก็บคนอุ่นด้วย Pilot
